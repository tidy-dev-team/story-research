# DropdownItem

The DropdownItem component represents individual items within a dropdown menu. It supports various content types including icons, severity badges, and checkboxes for multi-selection scenarios.

## Features

- **Rich Content**: Support for icons and severity badges
- **Interactive States**: Hover, selected, focused, and disabled states
- **Multi-Selection**: Checkbox support for multi-select dropdowns
- **Accessibility**: Full keyboard navigation and ARIA support
- **RTL Support**: Right-to-left layout compatibility
- **Size Variants**: Small, medium, and large sizes

## Styling Guidelines

### Color System

The DropdownItem follows a consistent color system:

- **Default Text**: `#FFFFFF` (white)
- **Selected Text**: `#0093EE` (primary blue)
- **Disabled Text**: `rgba(255, 255, 255, 0.38)` (38% opacity white)
- **Hover Background**: `rgba(255, 255, 255, 0.08)` (8% opacity white)
- **Selected Background**: `rgba(0, 147, 238, 0.12)` (12% opacity primary blue)

### Severity Badge Colors

- **Low**: Green (`#10B981` background, `#34D399` text)
- **Medium**: Yellow (`#F59E0B` background, `#FBBF24` text)  
- **High**: Orange (`#EA580C` background, `#FB923C` text)
- **Critical**: Red (`#DC2626` background, `#F87171` text)

### Spacing and Layout

- **Small**: 8px horizontal padding, 4px vertical padding
- **Medium**: 12px horizontal padding, 8px vertical padding
- **Large**: 16px horizontal padding, 12px vertical padding
- **Gap between elements**: 8px (small/medium), 12px (large)

## Accessibility Features

- Proper semantic markup with `role="option"`
- ARIA selected state with `aria-selected`
- Keyboard navigation support (Enter, Space)
- Focus management with visible focus indicators
- Screen reader compatible structure

## Best Practices

1. **Consistent Iconography**: Use the same icon style and size across all items
2. **Meaningful Labels**: Ensure labels clearly describe the item's purpose
3. **Logical Severity Mapping**: Use severity levels consistently across your application
4. **Performance**: Avoid complex icons or heavy images that could slow rendering
5. **Accessibility**: Always provide meaningful text labels, even when using icons

## Implementation Notes

- The component uses React.forwardRef for proper ref forwarding
- Event handlers are optimized to prevent unnecessary re-renders
- The component is fully controlled - selection state is managed by the parent
- All interactive elements use proper semantic HTML for accessibility

## Usage Patterns

### Simple Text Items
```typescript
const item = {
  id: "1",
  label: "Simple Option",
  value: "simple"
};
```

### Items with Icons
```typescript
const item = {
  id: "2", 
  label: "Settings",
  value: "settings",
  icon: <SettingsIcon />
};
```

### Items with Severity
```typescript
const item = {
  id: "3",
  label: "Critical Issue",
  value: "critical",
  severity: { level: "critical", text: "Critical" }
};
```

### Disabled Items
```typescript
const item = {
  id: "4",
  label: "Disabled Option", 
  value: "disabled",
  disabled: true
};
```
