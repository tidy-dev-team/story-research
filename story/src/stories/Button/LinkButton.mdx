# Link Button Component

The Link Button component is designed for navigation, styled as an underlined text link. It provides a clear visual cue that it will direct the user to another page or resource.

## Features

- Underlined text for clear affordance as a link.
- Standard link colors: blue for idle, lighter blue for hover.
- Support for leading and trailing icons.
- RTL (right-to-left) text direction support.
- Accessible focus states (ring).
- Disabled state (text color changes, underline removed, non-interactive).

## Import

```tsx
import { LinkButton } from './path/to/LinkButton';
import LanguageIcon from '@mui/icons-material/Language'; // Example Icon
```

## Basic Usage

```tsx
// Basic link button
<LinkButton label="Learn more" href="/about" />

// Opening in a new tab
<LinkButton 
  label="Visit Our Blog"
  href="/blog"
  target="_blank"
  rel="noopener noreferrer"
/>
```

## With Icons

Link buttons can include leading and/or trailing icons to provide additional context or visual appeal.

```tsx
// With leading icon
<LinkButton 
  label="Company Website"
  href="https://example.com"
  leadingIcon={<LanguageIcon sx={{ fontSize: "14px" }} />}
/>

// With trailing icon
<LinkButton 
  label="Next Article"
  href="/articles/next"
  trailingIcon={<ArrowForwardIcon sx={{ fontSize: "14px" }} />}
/>
```

## States

### Disabled

When disabled, the link button is non-interactive, its text color changes, and the underline is removed.

```tsx
<LinkButton label="Cannot Navigate" href="/restricted" disabled />
```

### Focus

The link button shows a focus ring when it receives keyboard focus or when the `focused` prop is true.

```tsx
<LinkButton label="Focused Link" href="/features" focused />
```

## RTL Support

For right-to-left languages, set the `rtl` prop to `true`. Icons will also be flipped accordingly.

```tsx
<LinkButton 
  label="קישור לדוגמה" // Example RTL text
  href="/rtl-page"
  rtl={true} 
  leadingIcon={<LanguageIcon sx={{ fontSize: "14px" }} />}
/>
```

## Event Handling

While primarily for navigation via `href`, an `onClick` handler can be provided for additional actions, like analytics tracking. Note that if `disabled` is true, `onClick` will not be triggered and navigation will be prevented.

```tsx
<LinkButton 
  label="Trackable Link"
  href="/tracked-page"
  onClick={(e) => {
    // e.preventDefault(); // Call this if you want to stop navigation and handle it manually
    console.log('LinkButton clicked, navigating to:', e.currentTarget.href);
    // trackAnalyticsEvent('link_click', { url: e.currentTarget.href });
  }} 
/>
```

## Accessibility

- Uses a standard `<a>` tag for semantic correctness.
- `aria-disabled` is applied when the `disabled` prop is true.
- Focus states are clearly visible.
- Underline provides a strong visual cue for its link nature.

## Design Considerations

- Use LinkButtons for navigational purposes where a standard hyperlink appearance is desired.
- The underline is a key part of its design, differentiating it from TextButton which is for actions.
- Ensure the `href` prop is always provided for proper link functionality.
