# Icon Button Component

The Icon Button component is a compact button that displays only an icon. It's ideal for toolbar actions, UI controls where space is limited, or for actions where the icon is universally understood.

## Features

- Multiple visual styles: Primary, Secondary, and Ghost
- Four size options: xs, s, m, and l
- Simple icon display without text
- Accessible focus states
- Disabled state
- Flexible icon passing methods

## Import

```jsx
import { Button, ButtonSize, ButtonType } from './path/to/IconButton';
import SettingsIcon from '@mui/icons-material/Settings';
```

## Basic Usage

There are two ways to use this component:

### Method 1: Pass the icon component directly (recommended)

```jsx
// Simple usage
<Button icon={SettingsIcon} />

// Primary button (default)
<Button 
  icon={SettingsIcon}
  type={ButtonType.Primary}
  size={ButtonSize.Medium} 
/>

// Secondary button
<Button 
  icon={HeadphonesIcon}
  type={ButtonType.Secondary} 
/>

// Ghost button
<Button 
  icon={LanguageIcon}
  type={ButtonType.Ghost} 
/>
```

### Method 2: Pass a pre-rendered icon element

```jsx
// Pre-rendered icon element
<Button 
  icon={<SettingsIcon sx={{ fontSize: "inherit" }} />}
/>
```

## Size Variations

```jsx
// xs (20x20)
<Button icon={SettingsIcon} size={ButtonSize.XSmall} />

// s (24x24)
<Button icon={SettingsIcon} size={ButtonSize.Small} />

// m (32x32, default)
<Button icon={SettingsIcon} size={ButtonSize.Medium} />

// l (40x40)
<Button icon={SettingsIcon} size={ButtonSize.Large} />
```

## States

### Disabled

```jsx
<Button icon={SettingsIcon} disabled />
```

### Focus

The button shows a focus ring when it receives keyboard focus or when the `focused` prop is true.

```jsx
<Button icon={SettingsIcon} focused />
```

## Event Handling

```jsx
<Button 
  icon={SettingsIcon}
  onClick={(e) => {
    console.log('Icon button clicked', e);
    // Handle the click event
  }} 
/>
```

## Accessibility

- When using icon-only buttons, consider adding a title or aria-label for screen readers
- Focus states are clearly visible
- Icons should be recognizable and have sufficient contrast

```jsx
// With accessibility label
<Button 
  icon={SettingsIcon}
  aria-label="Settings" 
  title="Settings"
/>
```

## Design Considerations

- Use icon buttons when space is limited or when the icon meaning is universally understood
- For actions that might be unclear from just an icon, consider using a regular Button with text
- Maintain consistency in icon usage throughout your application
- Use appropriate sizing based on touch targets for mobile interfaces
- Consider using tooltips for icon-only buttons to clarify their purpose
